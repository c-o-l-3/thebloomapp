# Email Authoring Workflow

This document describes the standard process for creating and editing email touchpoints across all Bloom clients.

## The Stack

| Tool | Role |
|------|------|
| LLM (Claude, GPT-4, etc.) | Generate initial email design as Unlayer JSON |
| Unlayer (react-email-editor) | WYSIWYG visual editing |
| Bloom visual editor | Import JSON, edit, save, export |
| GoHighLevel | Final delivery — paste exported HTML |

---

## The Workflow

### 1. Set Up Client Email Assets

Each client has an `email-generation/` directory under `clients/{client-name}/`:

```
clients/{client-name}/
└── email-generation/
    ├── system-prompt.md     # LLM system prompt (brand + schema)
    ├── WORKFLOW.md          # Client-specific notes
    └── emails/              # Unlayer design JSON files
        ├── 01-email-name.json
        └── ...
```

The `system-prompt.md` contains:
- Full Unlayer JSON schema for all block types
- Client brand (colors, fonts, voice, contact info)
- Hosted image URLs
- GHL merge tags
- Standard email structure

### 2. Generate Email JSON

Copy the entire contents of the client's `system-prompt.md` into any LLM. Append a brief describing the specific email:

```
## Per-Email Brief

- Name: Email 02 — Day 2
- Subject: What to actually look for in a wedding venue
- Preheader: A checklist that saves couples from expensive mistakes
- Theme: [describe the message]
- CTA: Schedule a tour
```

The LLM returns a complete Unlayer design JSON object. This is valid input for `loadDesign()`.

### 3. Import into the Visual Editor

1. In the Bloom app, open the journey and click a touchpoint
2. Click **"Edit Visually"** to open the Unlayer editor
3. Click **"Import JSON"** in the toolbar
4. Paste the LLM output and click **"Load Design"**

The email renders live in the canvas.

### 4. Edit Visually

- Click any text to edit copy inline
- Drag blocks from the left panel to add content
- Click images to swap them out
- For major structural changes, go back to the LLM

### 5. Save

Click **"Save"** — this writes both the Unlayer design JSON (for future editing) and compiled HTML to the database.

### 6. Export for GoHighLevel

Click **"Copy HTML"** → copy to clipboard → paste into the GHL email template editor.

## Unlayer JSON Format

The Unlayer design format is the source of truth for all emails. It is a JSON object that Unlayer's editor can load and re-export.

Top-level structure:
```json
{
  "body": {
    "id": "unique-id",
    "rows": [...],
    "values": {
      "backgroundColor": "#F9F7F4",
      "width": "600px",
      "preheaderText": "..."
    }
  }
}
```

Block types: `heading`, `text`, `image`, `button`, `divider`

See any client's `email-generation/system-prompt.md` for the full schema with examples.

---

## Database Storage

The `content` field on a touchpoint stores both representations:

```json
{
  "subject": "Email subject line",
  "preheaderText": "Preheader text",
  "body": "<compiled HTML — generated by Unlayer exportHtml()>",
  "unlayerDesign": { "body": { "rows": [...] } }
}
```

The `unlayerDesign` field is what gets loaded when you open the visual editor. The `body` HTML is what gets exported to GHL.

## Setting Up a New Client

1. Create `clients/{client-name}/email-generation/system-prompt.md`
   - Copy the structure from `clients/cameron-estate/email-generation/system-prompt.md`
   - Replace brand colors, fonts, voice, images, and contact info

2. Create `clients/{client-name}/email-generation/emails/` directory

3. Use the LLM workflow above to generate each email

4. Import each JSON into the Bloom visual editor and save

## GHL Merge Tags

Standard tags used across all clients (replaced by GHL at send time):

| Tag | Description |
|-----|-------------|
| `{{contact.first_name}}` | Recipient's first name |
| `{{links.book_tour}}` | Tour/appointment booking link |
| `{{links.unsubscribe}}` | Unsubscribe link (required in every email) |

Always preserve these exactly in generated JSON — do not substitute or modify them.

---

## Why Not MJML?

Some client directories contain an `email-factory/` folder with MJML templates. This is the legacy pipeline. It is no longer the active workflow because:

- Compiled MJML HTML cannot be round-tripped through a visual editor (inline styles overwrite everything)
- MJML requires a developer to make copy or layout changes
- Unlayer JSON supports both LLM generation and human visual editing in the same format

The `email-factory/` directories are kept for reference only.
